# \[ESX/QBCore/Qbox] Configuration

This resource is designed to be configured in-game.

Safezone Creator supports `ESX`, `QBCore`, and `Qbox`.

The default command for opening the admin menu is `/safezonemenu`.

Use the in-game menu to manage safezones, permissions, notifications, built-in bridge options, logs, icons, blips, and debug settings.

There is no normal Lua setup page for day-to-day configuration. Server owners should use the in-game admin menu instead of editing a config block.

From the in-game menu you can configure:

* safezones
* admin access
* notifications
* built-in bridge options
* inventory behavior
* zone engine behavior
* icons and blips
* debug options
* weapon whitelist behavior

If your staff can open the admin menu, they can complete the full Safezone Creator setup from there.
